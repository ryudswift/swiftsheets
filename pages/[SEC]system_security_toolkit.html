SYSTEM SECURITY TOOLKIT

An interactive guide to checking ports and vulnerabilities on your system.

STATUS: SECURE ANALYSIS MODE

1. BUILT-IN TERMINAL TOOLS

Tool: ss (Socket Statistics)

The modern tool for investigating sockets. Use it to quickly see what ports are open and which processes are listening on them. It has replaced the older `netstat` command.

ryud@sudev:~$ 
$ sudo ss -ltunp
You're getting a baseline inventory. Look for any surprising services in the `Process` column. If you see a process you don't recognize listening on a port, that's a red flag worth investigating. Also, check the `Local Address:Port` column to see if a service is exposed to the internet (`0.0.0.0` or `*`) or only to the local machine (`127.0.0.1`).

2. ESSENTIAL NETWORK SCANNER

Tool: nmap (Network Mapper)

The swiss-army knife for network exploration and security auditing. While `ss` tells you what your machine *thinks* is open, `nmap` tells you what is actually reachable from the network. It's essential for verifying your firewall rules and identifying services.

2a. Basic Port Scan

ryud@sudev:~$ 
$ nmap localhost
This command confirms which ports are actually open from a network perspective. Compare this list to the `ss` output. If a port is open here that you didn't expect, your firewall might not be configured correctly.

2b. Service & Version Detection

ryud@sudev:~$ 
$ sudo nmap -sV localhost
Here you're looking for outdated software versions. An attacker doesn't just see "port 5432," they see "PostgreSQL 9.5." They can then look for exploits for that specific version. Your goal is to find these old versions and update them.

2c. Vulnerability Scan

ryud@sudev:~$ 
$ sudo nmap --script vuln localhost
Look for any output labeled with "VULNERABLE". This command actively (but safely) tests for known flaws. If it finds something, it will often give you a CVE number (e.g., CVE-2022-12345) which you can search online to understand the risk and how to fix it.

3. UPGRADED & THIRD-PARTY TOOLS

For deeper and more automated analysis, you can use specialized open-source tools. These are often more powerful and provide more detailed reports.

Greenbone (OpenVAS)

A full-featured vulnerability scanner with a web interface. It maintains a huge, daily-updated feed of network vulnerability tests.

OWASP ZAP

The world's most popular free web application security scanner. Essential if any of your open ports are serving web pages or APIs.

Official Site →

Trivy

A simple and comprehensive scanner for vulnerabilities in container images, filesystems, and Git repos. Since you use Docker, this is a must-have.

Nuclei

A fast, template-based vulnerability scanner. It uses a large community-curated set of templates to send requests and check for thousands of known issues.
